# RAG 概述

检索增强生成（Retrieval-Augmented Generation，RAG）是一种结合了信息检索和大语言模型生成能力的技术架构。

## 为什么需要 RAG

大语言模型存在以下局限性：

1. **知识截止日期**：模型训练数据有截止日期，无法获取最新信息
2. **幻觉问题**：模型可能生成看似合理但错误的内容
3. **领域知识不足**：通用模型缺乏特定领域的专业知识

RAG 通过检索外部知识来增强 LLM 的回答，有效解决以上问题。

## RAG 工作流程

### 1. 文档处理（Document Processing）

```
原始文档 → 文本分割 → 向量化 → 存入向量数据库
```

关键步骤：
- 文档格式解析（PDF、Word、HTML等）
- 文本清洗和标准化
- 合理分块（Chunking）
- 选择合适的嵌入模型

### 2. 向量检索（Vector Retrieval）

```
用户查询 → 向量化 → 相似度搜索 → 检索相关文档
```

常用向量数据库：
- **Milvus**：开源、高性能
- **Pinecone**：托管服务、易用性好
- **Weaviate**：原生支持混合搜索
- **Chroma**：轻量级、开发友好

### 3. 生成（Generation）

```
检索结果 + 用户查询 → 提示词组装 → LLM 生成
```

提示词模板：
```
基于以下参考资料回答用户问题。如果资料中没有相关信息，请如实说明。

参考资料：
{retrieved_context}

用户问题：{user_question}

回答：
```

## 分块策略

### 固定大小分块
```python
chunk_size = 500
chunk_overlap = 50
```

### 语义分块
根据文档的自然结构（段落、章节）进行分割

### 递归字符分块
按层级尝试不同的分隔符（段落→句子→单词）

## 优化策略

### 混合搜索
结合语义搜索和关键词搜索（BM25）：

```
final_results = 0.7 * semantic_results + 0.3 * keyword_results
```

### 重排序（Reranking）
使用更精准的模型对检索结果进行重排序

### 查询扩展
将用户问题扩展为多个相关查询，提高召回率

### 上下文压缩
在发送给 LLM 前对检索结果进行压缩和摘要

## 评估指标

1. **召回率（Recall）**：相关文档被检索出来的比例
2. **精确率（Precision）**：检索结果中相关文档的比例
3. **MRR（Mean Reciprocal Rank）**：第一个相关文档的排名倒数均值
4. **NDCG**：考虑位置的相关性评估

RAG 是构建企业级 AI 应用的核心技术，掌握其原理和优化方法对于 AI 开发者至关重要。
