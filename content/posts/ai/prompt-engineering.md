# 提示工程简介

提示工程是与大语言模型交互的艺术，通过精心设计的提示词来获得更好的输出结果。

## 基础原则

### 1. 清晰具体的指令

❌ 不好的示例：
```
写一篇文章
```

✅ 好的示例：
```
请写一篇关于人工智能在医疗领域应用的技术博客，要求：
- 篇幅 1500-2000 字
- 包含至少 3 个实际案例
- 使用专业但易懂的语言
- 适合有一定技术背景的读者
```

### 2. 结构化输出

明确指定输出的格式和结构：
- 使用 Markdown 格式
- 使用表格或列表
- 指定 JSON 格式输出

## 高级技巧

### 零样本提示（Zero-shot）

直接给出任务描述，不需要示例：
```
将以下英文句子翻译成中文：
"The best way to predict the future is to create it."
```

### 少样本提示（Few-shot）

提供几个示例帮助模型理解：
```
将形容词转换为反义词。

示例：
happy -> sad
big -> small
fast -> slow
hot -> ____
```

### 思维链（Chain of Thought）

让模型展示推理过程：
```
问题：小明有10个苹果，送给朋友5个，又买了8个，现在有多少个？
解题步骤：
1. 小明开始有10个苹果
2. 送给朋友5个，10 - 5 = 5个
3. 又买了8个，5 + 8 = 13个
答案是：13个
```

## 实用模式

### 角色扮演
```
你是一位资深软件架构师，有15年的分布式系统设计经验。请分析以下系统架构的问题...
```

### 分步执行
```
第一步：理解用户需求
第二步：设计数据库结构
第三步：编写核心代码
请按以上步骤完成电商后台系统的设计
```

### 迭代优化
```
请为上述方案提供至少3个可以进一步优化的地方，并说明原因。
```

## 常见陷阱

1. **提示词过于模糊** → 分解任务，逐步引导
2. **一次性要求太多** → 拆分成多个小任务
3. **忽略上下文限制** → 注意 token 数量限制
4. **不验证输出** → 始终对关键信息进行校验

掌握提示工程技巧可以显著提升 LLM 的使用效率，是每个 AI 应用开发者必备的技能。
