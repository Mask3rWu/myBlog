# 计算机网络概述

计算机网络是现代信息技术的基石，它使得全球范围内的数据交换和通信成为可能。

## 网络分类

### 按地理范围划分

| 类型 | 范围 | 典型应用 |
|------|------|----------|
| PAN | 几米内 | 蓝牙设备 |
| LAN | 局部区域 | 办公室网络 |
| MAN | 城市范围 | 城域光纤 |
| WAN | 国家/全球 | 互联网 |

### 按拓扑结构划分

```
星型拓扑：中心节点+多个终端（易于管理，中心故障全网瘫痪）
总线型：共用一根主干线（成本低，冲突严重）
环型：节点形成闭环（令牌环网络）
网状型：节点多路径连接（高可靠性，配置复杂）
树型：分层星型结构（层次化管理）
```

## OSI 七层模型

OSI（开放系统互联）模型是网络通信的标准框架：

### 1. 物理层（Physical）

处理比特的传输：
- 传输介质：双绞线、光纤、无线
- 编码方式：曼彻斯特编码、差分编码
- 比特同步：时钟恢复
- 传输速率：100Mbps、1Gbps、10Gbps

### 2. 数据链路层（Data Link）

提供可靠的链路传输：
- 帧封装与同步
- MAC 地址寻址
- 差错检测：CRC 循环冗余校验
- 流量控制：滑动窗口协议
- 交换机工作原理

### 3. 网络层（Network）

实现跨网络的数据转发：
- IP 协议：逻辑寻址与路由
- 路由算法：距离矢量、链路状态
- ICMP：网络控制报文
- NAT：网络地址转换
- IPv4 vs IPv6

```python
# IP 地址分类
class IPv4Address:
    CLASS_A = (0b0, 126, "255.0.0.0")      # 0.0.0.0 - 126.255.255.255
    CLASS_B = (0b10, 128, "255.255.0.0")   # 128.0.0.0 - 191.255.255.255
    CLASS_C = (0b110, 192, "255.255.255.0") # 192.0.0.0 - 223.255.255.255
    CLASS_D = (0b1110, 224, "多播地址")      # 224.0.0.0 - 239.255.255.255
    CLASS_E = (0b1111, 240, "保留地址")      # 240.0.0.0 - 255.255.255.255
```

### 4. 传输层（Transport）

提供端到端的可靠传输：

**TCP 协议核心机制**

```python
# TCP 三次握手
# 客户端                    服务器
#   |                          |
#   |------- SYN=1, seq=x ---->|
#   |<----- SYN=1, ACK=1, ----|
#   |       seq=y, ack=x+1    |
#   |------ ACK=1, ack=y+1 -->|
#   |         连接建立         |
```

| 特性 | 说明 |
|------|------|
| 可靠传输 | 确认机制、重传机制 |
| 流量控制 | 滑动窗口、拥塞窗口 |
| 拥塞控制 | 慢启动、拥塞避免、快恢复 |

**UDP 协议特点**
- 无连接：不需要握手
- 不可靠：不保证交付
- 高效：头部开销小（8字节 vs 20字节）
- 适用场景：实时音视频、DNS、QUIC

### 5. 会话层（Session）

管理通信会话：
- 会话建立与维护
- 同步点设置
- 对话控制

### 6. 表示层（Presentation）

数据格式转换：
- 编码转换：ASCII、UTF-8、Unicode
- 压缩解压：GZIP、DEFLATE
- 加密解密：SSL/TLS

### 7. 应用层（Application）

面向用户的网络服务：

```
HTTP/HTTPS：万维网服务
FTP：文件传输
SMTP/POP3/IMAP：电子邮件
DNS：域名解析
SSH/Telnet：远程登录
SNMP：网络管理
```

## 核心协议详解

### HTTP 协议演进

```
HTTP/1.0 → HTTP/1.1 → HTTP/2 → HTTP/3
```

| 版本 | 关键特性 |
|------|----------|
| HTTP/1.1 | 持久连接、管道化 |
| HTTP/2 | 多路复用、头部压缩、服务器推送 |
| HTTP/3 | QUIC协议、0-RTT、快速连接 |

### DNS 域名系统

```
用户输入域名 → 本地DNS缓存 → 递归DNS服务器 → 根域名服务器 → 顶级域名服务器 → 权威DNS服务器
```

### TLS/SSL 加密

```
ClientHello → ServerHello → 证书验证 → 密钥协商 → 加密通信
```

## 网络安全基础

### 常见攻击类型

1. **DDoS**：分布式拒绝服务攻击
2. **MITM**：中间人攻击
3. **SQL注入**：数据库攻击
4. **XSS**：跨站脚本攻击
5. **CSRF**：跨站请求伪造

### 防御措施

- 防火墙（包过滤、状态检测、应用网关）
- 入侵检测/防御系统（IDS/IPS）
- VPN 虚拟专用网络
- 零信任架构
